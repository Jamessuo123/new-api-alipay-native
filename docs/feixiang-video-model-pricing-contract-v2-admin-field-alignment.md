# FeiXiangApi 视频模型定价契约 v2：后台字段与前端展示对齐

> Marker: FEIXIANG_VIDEO_MODEL_PRICING_CONTRACT_V2_ADMIN_FIELD_ALIGNMENT

本版本目标：让 Seedance 视频模型的后台字段、模型广场卡片、模型详情页和分组倍率显示使用同一套口径，后续改价时不需要再到前端硬编码里找价格。

## 1. 现阶段不改数据库结构

为了避免影响线上计费，本版本不新增表、不改字段类型，仍复用 NewAPI 现有模型定价字段。

## 2. 字段含义映射

| 后台字段 | 视频模型语义 | 前端展示 | 备注 |
| --- | --- | --- | --- |
| 模型倍率 / 输入价格 | 文生视频 / 无视频输入单价 | 无视频输入、视频生成单价 | Seedance 卡片和详情页默认展示这个字段。 |
| 图像输入倍率 / 图像输入价格 | 图生视频 / 含视频输入单价 | 含视频输入 | 如后台当前版本没有该字段，则详情页使用兼容兜底。 |
| 补全倍率 / 输出价格 | 输出视频价格 | 输出视频已包含 | Seedance 输出视频已包含在生成单价里，建议填 0。 |
| 分组倍率 | 当前用户最终计费分组倍率 | default / 业务分组最终价 | 卡片价格和卡片底部分组名必须一致。 |
| 模型 ID 后缀 | 分辨率 | 480p / 720p / 1080p / 4K | 例如 seedance-2.0-1080p。 |

## 3. 前端一致性原则

1. 模型广场卡片不再显示和底部分组不一致的价格。
2. 若卡片按某个业务分组计费，底部就显示该业务分组名称。
3. 若卡片按 default 计费，底部就显示 default 分组。
4. 详情页的分组对比表只做展示，不反向影响计费。
5. 不主动测试模型，不产生额外费用。

## 4. 后续 v3 建议

当需要彻底产品化视频定价时，可以新增正式字段：

```json
{
  "billing_type": "video_generation",
  "resolution": "1080p",
  "text_to_video_price": 11.36,
  "video_input_price": 6.91,
  "output_included": true,
  "provider_model_id": "doubao-seedance-2-0-260128"
}
```

但这会涉及数据库迁移、后台表单、API DTO 和前端渲染的完整升级，不建议和当前线上价格修复混在一起做。
